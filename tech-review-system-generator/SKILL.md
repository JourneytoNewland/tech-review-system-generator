# 技术方案评审系统生成器

根据技术方案文档快速生成功能完整的评审系统。

## 快速开始

```
输入：技术方案文档路径（如 ./REQ.md）
输出：index.html - 可直接在浏览器中打开的评审系统
```

## 执行流程

### 1. 分析文档，提取结构

读取文档，识别主要章节和技术点：
- 提取章节标题和层级关系
- 识别关键技术决策点
- 确定评审模块数量（通常 4-8 个）

### 2. 生成评审模块

根据文档内容生成评审模块，使用 [模块映射规则](references/module-mapping.md)。

### 3. 创建评审项

为每个模块生成 5-30 个评审项（根据粒度配置）：
- 提取技术点作为评审项标题
- 从文档提取详细描述
- 根据类型生成检查项（见 [检查项模板](references/checklist-templates.md)）
- 设置优先级（P0/P1/P2）和分类标签
- **识别疑问点**：对理解不清或逻辑有疑问的内容，生成"疑问澄清类"评审项

### 3.1 疑问点识别规则

在分析文档时，对以下情况生成澄清类评审项：

1. **信息缺失**：关键技术细节未说明
2. **逻辑矛盾**：前后描述不一致
3. **选型未说明**：技术选型缺少理由
4. **性能不合理**：指标与方案不匹配
5. **依赖不明确**：外部依赖未列出
6. **边界未考虑**：异常场景未处理

### 4. 生成 HTML 系统

生成包含以下内容的单文件 HTML：
- React + ReactDOM + Babel (CDN)
- Tailwind CSS (CDN)
- Chart.js (CDN)
- FontAwesome (CDN)
- 完整的评审系统代码

## 配置选项

详细的配置选项参见 [配置指南](references/config.md)。

**快速配置**：

| 参数 | 选项 | 默认值 |
|------|------|--------|
| 内容粒度 | concise/standard/detailed | standard |
| 界面风格 | modern/clean/technical | modern |
| 可视化 | yes/no | yes |
| 系统名称 | 自定义字符串 | 自动生成 |

## 输出结构

生成的系统包含：
- **Dashboard**：概览页面，包含 3 个可视化图表
- **模块视图**：每个模块的详细评审页面
- **评审项卡片**：可展开/折叠，包含检查清单和评论
- **导航**：左侧可折叠导航栏
- **导出**：JSON 格式报告

## 最佳实践

1. **文档分析优先**：充分理解技术方案后再划分模块
2. **模块聚焦**：每个模块聚焦一个技术领域
3. **评审项具体**：每个评审项应有明确验收标准
4. **检查项适量**：3-6 个检查项，覆盖核心技术点
5. **优先级清晰**：P0 核心关键、P1 重要功能、P2 优化改进
